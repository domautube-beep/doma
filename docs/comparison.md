# DOMA vs 다른 Claude Code 시스템

## 기본 Claude Code vs DOMA Brain

| 기능 | 기본 Claude Code | DOMA Brain |
|------|-----------------|------------|
| 세션 간 기억 | ❌ 매번 새로 시작 | ✅ 자동 축적 (Memory + Sessions) |
| 프로젝트 브리핑 | ❌ | ✅ 세션 시작 시 자동 |
| 세션 기록 | ❌ | ✅ "끝" 한 마디로 6곳 저장 |
| 전문 에이전트 | ❌ 범용 1개 | ✅ 77개 전문 에이전트 |
| 디자인 시스템 | ❌ | ✅ 5개 디자인 스킬 |
| 작업 라우팅 | ❌ 수동 | ✅ 자동 스킬/에이전트 매칭 |
| 지식 축적 | ❌ | ✅ HBM 계층 + Pipe 교차 연결 |
| 토큰 절약 | ❌ | ✅ 모델 자동 선택 (sonnet/opus) |

**핵심 차이**: 기본 Claude Code는 대화가 끝나면 모든 맥락이 사라진다. DOMA Brain은 세션이 끝날 때마다 Obsidian, Memory, Action Tracker 등 6곳에 자동 저장되어 다음 세션에서 바로 이어받는다.

## DOMA vs MemPalace

MemPalace는 단순 기억 저장(Memory) 특화 시스템이다.

| 기능 | MemPalace | DOMA Brain |
|------|-----------|------------|
| 기억 저장 방식 | 평면 파일 저장 | HBM 계층 (L0~L4) + 7개 Pipe |
| 지식 간 연결 | ❌ 개별 저장 | ✅ Pipe 교차 연결로 인사이트 생성 |
| 에이전트 시스템 | ❌ | ✅ 역할별 77개 전문 에이전트 |
| 워크플로우 자동화 | ❌ | ✅ 스킬 기반 자동 라우팅 |
| 프로젝트 관리 | ❌ | ✅ projects-registry + Action Tracker |
| 디자인 작업 지원 | ❌ | ✅ Supanova 4종 + design-system-builder |
| 텔레그램 연동 | ❌ | ✅ 모바일 원격 제어 + 아침 브리핑 |
| 세션 연속성 | 부분적 | ✅ 집컴/회사컴 Google Drive 동기화 |

**핵심 차이**: MemPalace는 "기억하는 AI"이고, DOMA Brain은 "기억하고 + 행동하는 AI 팀"이다. 지식을 저장하는 것에서 그치지 않고 Pipe 교차 연결로 새로운 인사이트를 자동 생성한다.

## DOMA vs awesome-claude-md

awesome-claude-md는 Claude를 잘 쓰기 위한 프롬프트 모음집(규칙 라이브러리)이다.

| 기능 | awesome-claude-md | DOMA Brain |
|------|-------------------|------------|
| 핵심 개념 | 좋은 CLAUDE.md 작성법 | 개인화된 AI 운영 체계 |
| 제공 형태 | 템플릿/예시 모음 | 실제 동작하는 스킬 + 에이전트 시스템 |
| 맞춤화 수준 | 범용 (누구나 사용 가능) | 특정 사용자 맞춤 (개인 데이터 축적) |
| 세션 기억 | ❌ 규칙만 있음 | ✅ Memory + Obsidian 실시간 동기화 |
| 에이전트 팀 | ❌ | ✅ setup-team으로 역할 자동 배분 |
| 지식베이스 연결 | ❌ | ✅ Obsidian HBM 연동 |
| 개인 해자 (경쟁 우위) | ❌ 누구나 같은 템플릿 | ✅ 축적할수록 본인만의 AI로 진화 |
| 스킬 실행 시스템 | ❌ | ✅ 30개+ 전문 스킬 즉시 실행 |

**핵심 차이**: awesome-claude-md는 "Claude에게 좋은 지시문 작성법"을 가르쳐주는 참고서다. DOMA Brain은 그 지시문을 실제 실행하는 엔진이다. 규칙을 아는 것과 그 규칙이 자동으로 동작하는 것의 차이.

## DOMA의 독창성 요약

### 1. HBM(Hermes Brain Map) 계층 구조
단순 폴더 정리가 아니라 L0(정체성) → L4(버퍼)까지 지식의 성숙도에 따라 계층화. 01-Pipe 7개(사업/디자인/AI무기/음악/인풋/전체맵/파일시스템)가 교차 연결되어 파편 지식이 새 아이디어로 변환된다.

### 2. 개인화 해자(Moat)
사용하면 사용할수록 본인만의 시스템이 된다. 피드백 Memory, 세션 히스토리, 프로젝트 결정 로그가 누적되어 타인이 복사해도 동일한 결과를 낼 수 없다.

### 3. 집컴/회사컴 연속성
Google Drive 심볼릭 링크로 어디서나 동일한 맥락으로 작업 시작. 세션이 끊겨도 "어디까지 했는지" 자동으로 이어받는다.

### 4. 모델 비용 최적화
탐색/리서치 → sonnet, 최종 실행/리팩토링 → opus. 자동 라우팅으로 토큰 비용을 30~50% 절감.
