## 8단계 마스터 프로세스

모든 디자인에 공통 적용. 매체(브랜드/웹/앱/카드뉴스/배너/패키지)에 따라 우선순위만 다르다.

### 0. Brand Discovery (브랜드 조사) — 반드시 먼저

디자인 전에 **그 브랜드를 알아야 한다**. 모르면 기본값으로 가게 되고, 그건 그 브랜드가 아니다.

**조사 순서:**
1. 브랜드명/회사명으로 웹 검색 → 공식 사이트, SNS, 기존 시각물 확인
2. 기존 디자인 가이드/브랜드북이 있으면 분석 (컬러, 서체, 톤, 로고 사용법)
3. 옵시디언 L3-Active/Work/ 또는 L1-Knowledge/에 관련 노트가 있으면 읽기
4. Figma MCP로 기존 시안이 있으면 `get_design_context`로 분석
5. 경쟁사/동종업계 시각 언어 벤치마크 (최소 3개)

**산출물:**
```
브랜드: {이름}
업종: {카테고리}
기존 컬러: {있으면 hex 코드, 없으면 "미정"}
기존 서체: {있으면 이름, 없으면 "미정"}
톤앤매너: {형용사 3개}
타깃: {누구}
벤치마크: {경쟁사 3개 + 시각적 특징}
```

이 산출물이 없으면 다음 단계로 넘어가지 않는다.

**{회사명} 프로젝트인 경우**: [[L2-Methods/Work/{회사명}-브랜드관리]] + CLAUDE.md의 디자인 시스템 섹션을 반드시 먼저 읽는다. 과목별 컬러, 은별색, 3D Geometry Pattern, Pretendard 등 기존 시스템을 따른다.

### 1. Experience Intent 정의

디자인은 색/폰트가 아니라 **경험 목표**부터. 이걸 안 정하면:
- 브랜드는 멋있지만 기억 안 남고
- 웹은 예쁘지만 전환이 안 나오고
- 배너는 튀지만 클릭이 안 된다

정의할 것:
- 이 디자인은 무엇을 하게 만들어야 하는가
- 사용자는 3초 안에 무엇을 이해해야 하는가
- 느껴야 할 감정 / 느끼면 안 되는 감정

### 2. Visual to Data

감각을 측정 가능한 언어로 바꾸는 단계:
- 컬러 토큰 (역할별: text/bg/surface/accent/success/error)
- 타이포 스케일 (Display 3.5rem ~ Caption 0.75rem)
- 간격 시스템 (8px 단위)
- 코너 반경, 선 두께, 그림자 깊이
- 이미지/아이콘 규칙

### 3. Structural Reverse Engineering

레퍼런스를 구조로 해체:
- 어떤 그리드인가
- 시선은 어디서 시작하나
- 정보 계층은 몇 단계인가
- 어떤 컴포넌트로 재사용 가능한가

### 4. Principle-based Guideline

가이드 문서 = 스타일가이드가 아니라 **판단 기준서**:
- **A. Principles** — 왜 이렇게 디자인하는가
- **B. Rules** — 무조건 지켜야 하는 것
- **C. Patterns** — 자주 반복되는 화면/요소 조합
- **D. QA** — 결과물 확인 체크리스트

### 5. Component System

디자인을 결과물이 아니라 **부품**으로 쪼개기:
- Atoms: 색, 글자, 여백, 라인, 아이콘
- Molecules: 버튼, 태그, 입력창, 캡션 블록
- Organisms: 카드, 네비게이션, 히어로, 비교표
- Templates: 랜딩, 상세, 카드뉴스 템플릿, 패키지 전개면

### 6. Production — HTML 생성

가이드 → HTML 코드 → 브라우저 오픈. 생성 지시문 구조:
- 목적, 타깃, 매체 역할, 핵심 메시지
- 반드시 지킬 규칙, 금지 요소
- 산출 포맷, 자기 검수 항목

### 7. Feedback Loop

초안 → 시각 검토 → 규칙 수정 → 재생성 → QA → 템플릿 업데이트

## 비주얼 퀄리티 기준

### 원칙
비주얼은 브랜드에서 나온다. 기본값(다크모드+골드)을 쓰지 않는다.

**Step 0에서 파악한 브랜드 톤앤매너가 비주얼의 출발점이다.**
- 프리미엄 브랜드 → 여백, 절제, 타이포 위계로 고급감
- 친근한 브랜드 → 컬러풀, 라운드, 일러스트
- 테크 브랜드 → 그리드, 모노스페이스, 깊이감
- 교육 브랜드 → 명확한 정보 위계, 신뢰감, 과목 컬러

### 공통 적용 (브랜드 무관)
- **웹폰트 필수**: 시스템 폰트만으로 끝내지 않는다
- **여백**: 과감하게. 빈 공간 = 호흡
- **정보 위계**: 한눈에 뭐가 중요한지 보여야 한다
- **한 화면 1메시지**: 다 넣으려 하지 않는다

### 브랜드가 없을 때 (신규 제작)
브랜드가 아직 없는 경우에만 아래 기본값을 시작점으로 사용:
- 본문: Pretendard
- 포인트: 브랜드 성격에 맞는 서체 1개 선택
- 컬러: 60/30/10 비율로 3색 이내
- 배경: 브랜드 톤에 맞게 (다크/라이트/컬러 중 선택)

### 절대 금지
- 브랜드 조사 없이 기본값(다크+골드)으로 만들기
- 모든 디자인에 같은 스타일 적용 (그라데이션, 메탈릭 등)
- 기본 시스템 폰트만 사용
- 장식이 콘텐츠를 가리는 것

## 분야별 우선순위

### 브랜드 디자인
> 정체성의 일관성과 판별력

**목적**: 기억되는가, 구별되는가, 신뢰를 주는가, 확장 가능한가

**반드시 정의**:
- Brand Core (미션/가치/차별점/포지션/한문장 정의)
- Brand Personality (차분함/대담함/지성적 등 + 말투/감정온도)
- Visual Identity (로고/컬러/서체/키비주얼/사진톤/그래픽모티프)
- Verbal Identity (슬로건/핵심메시지/금지표현/CTA톤)

**QA**: 이걸 보고 누구인지 바로 떠오르는가? 경쟁자와 바꿔 끼워도 이상하지 않을 정도로 평범하진 않은가?

**망하는 이유**: 로고만 만들고 끝남, 매체별 결과물이 다 따로놈

### 웹 디자인
> 이해와 전환

**목적**: 빠르게 이해 → 자연스럽게 행동 → 이탈 방지 → 신뢰 구축

**구조**: User Task Mapping → IA → Wireframe → Visual → Conversion Layer

**QA**: 첫 5초 안에 뭘 하는 서비스인지 보이는가? CTA가 한 개로 선명한가?

**망하는 이유**: 비주얼은 세련됐는데 정보구조 약함, PC 기준으로만 설계

### 앱 디자인
> 조작성과 반복 사용 편의

**목적**: 자주 써도 피곤하지 않고, 조작이 즉시 납득되고, 학습 비용이 낮은가

**핵심**: Flow Design → Interaction Rule → State Design → Repetition Comfort

**QA**: 처음 써도 길을 잃지 않는가? 한 손 사용에서 편한가?

**망하는 이유**: 웹처럼 디자인함, 상태 설계 없음, 빈 화면/오류 화면 미설계

### 카드뉴스
> 시퀀스 전달력과 저장가치

**목적**: 멈추게 → 넘기게 → 끝까지 보게 → 저장/공유하게

**구조**: Hook(첫장 멈춤) → Tension(왜 봐야 하나) → Breakdown(한장씩 소화) → Payoff(결론/행동유도)

**규칙**: 한 장 = 한 메시지, 텍스트량 제한, 시각 포인트 1개

**QA**: 첫 장에서 멈출 이유가 있는가? 다음 장이 궁금한가?

**망하는 이유**: 포스터처럼 만듦, 텍스트 과다, 장마다 역할 없음

### 배너
> 즉시 주목과 클릭

**목적**: 멈추기 → 한 줄 이해 → 클릭

**원칙**: One Message Rule, One Focus Rule, Speed First, Contrast First

**구성**: 핵심 헤드라인 + 짧은 서브카피 + 시각화 + CTA + 브랜드 식별자

**QA**: 1초 안에 무슨 배너인지 보이는가? 가장 중요한 단어가 눈에 꽂히는가?

**망하는 이유**: 다 넣으려 함, 헤드라인/제품/CTA가 서로 싸움

### 패키지
> 구매 순간의 선택성과 제품 경험

**목적**: 선택되기, 신뢰되기, 이해되기, 개봉 경험이 좋기

**구조**: Shelf Context → Front-of-Pack Hierarchy → Structural Design → Info Layer → Unboxing

**QA**: 매대에서 2초 안에 무슨 제품인지 보이는가? 경쟁 제품 옆에서 구분되는가?

**망하는 이유**: 전면에 다 넣음, 진열 환경 미고려, 예쁜데 생산성 나쁨

## 공통 실무 체크리스트

모든 디자인 작업 전/후에 확인:

```
[전략] 누구를 위한 디자인인가? 어떤 행동을 유도하는가?
[메시지] 한 줄 핵심 메시지가 있는가? 정보 우선순위가 정리됐는가?
[구조] 시선 흐름이 명확한가? 정보 위계가 분명한가?
[스타일] 브랜드와 연결되는가? 과한 장식이 없는가?
[행동] CTA가 자연스러운가? 무엇을 해야 하는지 보이는가?
[생산] 템플릿화 가능한가? AI/팀원이 재현 가능한가?
[검수] 3초 안에 이해되는가? 경쟁물과 구분되는가? 실제 환경에서 무너지지 않는가?
```

## 자기 검수

```
□ Step 0 Brand Discovery를 했는가 (브랜드 조사 없이 만들지 않았는가)
□ 그 브랜드의 기존 컬러/서체/톤을 반영했는가
□ Experience Intent에 맞는가
□ 여백이 충분한가
□ 한 화면 1메시지인가
□ 장식이 콘텐츠를 방해하지 않는가
□ 분야별 QA 질문을 통과하는가
□ 빼도 되는 요소가 있으면 뺐는가
□ 다크모드/그라데이션을 기본값으로 쓰지 않았는가 (브랜드가 요구할 때만)
```
