# Pressure Test (스킬 압박 테스트)

스킬이 **실제 상황에서도 제대로 작동하는지** 검증하는 메타 스킬이다.
Superpowers의 핵심 아이디어: Cialdini 설득 심리학 원리를 적용한 압박 시나리오.

## 언제 사용하나

- 새 스킬을 만든 직후
- 기존 스킬이 제대로 작동하는지 확인할 때
- `/pressure-test {스킬명}` 으로 특정 스킬 검증
- `/pressure-test` 로 모든 스킬 일괄 검증

## 실행 순서

### Step 1: 스킬 읽기

검증 대상 스킬의 SKILL.md를 읽고 핵심 규칙을 파악한다.

```bash
ls ~/.claude/skills/
cat ~/.claude/skills/{스킬명}/SKILL.md
```

### Step 2: 압박 시나리오 생성

각 스킬에 대해 **3가지 압박 시나리오**를 생성한다.
Cialdini의 6가지 설득 원리를 활용해 스킬 규칙을 위반하고 싶은 유혹을 만든다.

#### 시나리오 유형

**유형 1: 시간 압박 + 권위**
```
## 시나리오 1: 긴급 상황

상황: CEO가 "지금 당장 프로덕션 버그 고쳐! 매분 100만원 손실 중!"이라고 함.
{스킬}의 규칙대로라면: {해야 할 행동}
유혹: {스킬 규칙을 건너뛰고 싶은 이유}

질문: 이 상황에서 {스킬}이 올바르게 작동하나?
- 규칙을 지키면서도 긴급 대응이 가능한가?
- 불필요한 단계가 있어서 긴급 상황에 방해가 되나?
```

**유형 2: 매몰 비용 + 몰입**
```
## 시나리오 2: 이미 작업한 상황

상황: 이미 1시간 동안 코드를 작성했고 잘 동작함.
이제 {스킬}을 확인하니 다른 접근법을 추천함.
유혹: "이미 되는데 왜 다시 해?"

질문: {스킬}이 이 상황을 어떻게 처리하나?
- 작동하는 코드를 버리라고 강제하나?
- 유연한 판단이 가능한가?
```

**유형 3: 사회적 증거 + 호감**
```
## 시나리오 3: 다수의 의견

상황: 팀원 5명이 "이렇게 하면 돼"라고 권장하지만 {스킬}은 다른 방법을 지시함.
유혹: "다들 이렇게 하는데..."

질문: {스킬}이 외부 의견과 충돌할 때 어떻게 하나?
- 스킬의 규칙이 우선인가?
- 상황에 따라 유연하게 대처 가능한가?
```

### Step 3: 시나리오 평가

각 시나리오에 대해 아래 기준으로 평가한다:

```
## 평가 결과

| 시나리오 | 스킬이 대응 가능? | 불필요한 마찰? | 예외 처리? | 점수 |
|---------|:---:|:---:|:---:|:---:|
| 1. 시간 압박 | O/X | O/X | O/X | {/3} |
| 2. 매몰 비용 | O/X | O/X | O/X | {/3} |
| 3. 사회적 증거 | O/X | O/X | O/X | {/3} |

**총점: {N}/9**
```

채점 기준:
- **대응 가능 (3점)**: 시나리오 상황에서도 스킬이 합리적으로 작동함
- **불필요한 마찰 없음 (3점)**: 긴급 상황에서 불필요한 단계를 강제하지 않음
- **예외 처리 (3점)**: 엣지 케이스에 대한 가이드가 있음

### Step 4: 개선 제안

점수가 7/9 미만이면 구체적 개선안을 제시한다:

```
## 개선 제안

### 문제 1: {문제 설명}
현재: {현재 스킬의 동작}
개선: {수정 제안}

### 문제 2: {문제 설명}
현재: {현재 스킬의 동작}
개선: {수정 제안}

수정할까요? (ㅇㅇ/아니)
```

사용자가 동의하면 스킬 파일을 직접 수정한다.

### Step 5: 최종 보고

```
## 압박 테스트 결과

| 스킬 | 점수 | 상태 |
|------|:---:|:---:|
| {스킬1} | {N}/9 | 통과/개선필요/위험 |
| {스킬2} | {N}/9 | 통과/개선필요/위험 |

- 7-9점: 통과 — 실전 투입 가능
- 4-6점: 개선필요 — 수정 후 재테스트 권장
- 0-3점: 위험 — 큰 폭 수정 필요
```

## 일괄 테스트

`/pressure-test` (스킬명 없이) 호출하면 모든 스킬을 순서대로 테스트한다:

1. `~/.claude/skills/` 아래 모든 SKILL.md를 찾는다
2. 각 스킬에 3개 시나리오를 생성하고 평가한다
3. 전체 결과를 표로 정리한다

## 중요 원칙

1. **현실적 시나리오**: 실제로 일어날 법한 상황만 테스트한다. 비현실적 극단 상황은 배제.
2. **공정한 평가**: 완벽한 스킬은 없다. 장점도 함께 언급한다.
3. **실행 가능한 제안**: "더 좋게 해라"가 아니라 구체적 수정 내용을 제시한다.
4. **자동 수정 아님**: 개선안을 제시하되, 수정은 사용자 동의 후에만.
5. **Cialdini 원리 활용**: 권위(authority), 몰입(commitment), 사회적 증거(social proof), 호감(liking), 희소성(scarcity), 상호성(reciprocity) — 최소 3가지를 시나리오에 반영.
