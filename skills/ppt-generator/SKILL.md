# PPT Generator

발표 주제, 대상 청중, 발표 시간을 입력받아 완성된 .pptx 파일을 생성하는 스킬이다.

## 입력 수집

사용자에게 다음 3가지를 확인한다 (이미 제공된 건 다시 묻지 않는다):

1. **발표 주제** — 무엇에 대한 발표인지
2. **대상 청중** — 누구에게 발표하는지 (경영진, 학생, 개발자, 투자자 등)
3. **발표 시간** — 몇 분짜리 발표인지

선택 사항 (제공되면 반영):
- 테마: `professional`(기본), `dark`, `warm`, `minimal`
- 발표자 이름, 날짜
- 특별히 포함해야 할 내용

## 슬라이드 수 산정

발표 시간 기준으로 슬라이드 수를 자동 산정한다:

| 발표 시간 | 슬라이드 수 (표지/마무리 포함) |
|----------|--------------------------|
| 5분      | 6~8장                     |
| 10분     | 10~13장                   |
| 15분     | 14~18장                   |
| 20분     | 18~22장                   |
| 30분     | 25~30장                   |

핵심 원칙: **슬라이드 1장 = 1~1.5분**. 빽빽한 슬라이드보다 여유 있는 구성이 낫다.

## 슬라이드 구성 규칙

### 필수 구조
```
1. 표지 (title_slide)
2. 목차 (agenda_slide) — 10분 이상일 때
3~N. 본문 슬라이드 (content_slides)
마지막. 마무리 (closing_slide)
```

### 각 슬라이드에 반드시 포함할 요소

| 요소 | 설명 |
|------|------|
| **제목** (`title`) | 슬라이드의 핵심을 한 줄로 |
| **핵심 메시지** (`key_message`) | 이 슬라이드에서 청중이 가져가야 할 한 문장 |
| **본문** (`bullets`) | 3~5개 불릿 포인트 |
| **시각자료 지시사항** (`visual_description`) | 어떤 차트/이미지/다이어그램이 들어가야 하는지 구체적으로 기술 |
| **발표자 노트** (`notes`) | 이 슬라이드에서 말할 내용 (2~4문장) |

### 레이아웃 타입

슬라이드 내용에 따라 적절한 레이아웃을 선택한다:

- `text_only` — 텍스트 중심 (개념 설명, 정의)
- `text_and_visual` — 좌측 텍스트 + 우측 시각자료 (데이터, 프로세스)
- `two_column` — 비교/대조 (AS-IS vs TO-BE, 장단점)
- `visual_focus` — 시각자료 중심 (차트, 다이어그램, 아키텍처)

## 청중 맞춤 조정

청중에 따라 톤과 깊이를 조절한다:

- **경영진/임원**: 결론 먼저, 핵심 수치 강조, 의사결정 포인트 명확히
- **투자자**: 시장 기회, 차별화, 성장 지표 중심
- **개발자/엔지니어**: 기술 상세, 아키텍처 다이어그램, 코드 예시
- **학생/일반인**: 쉬운 용어, 비유 활용, 시각자료 많이
- **고객/파트너**: 가치 제안, 사례, ROI 중심

## 파일 생성 프로세스

### Step 1: 슬라이드 구성안 작성

먼저 사용자에게 전체 슬라이드 구성안을 보여준다:

```
## 슬라이드 구성안 (N장, M분)

1. [표지] 제목
2. [목차] ...
3. [본문] 제목 — 핵심메시지
...
N. [마무리] 감사합니다
```

사용자 승인 후 생성 진행한다.

### Step 2: JSON 데이터 생성

슬라이드 데이터를 JSON으로 구성한다. 구조:

```json
{
  "theme": "professional",
  "title_slide": {
    "title": "발표 제목",
    "subtitle": "부제",
    "presenter": "발표자",
    "date": "2026-03-21",
    "notes": "인사 및 발표 소개"
  },
  "agenda_slide": {
    "title": "목차",
    "items": ["항목1", "항목2", "항목3"],
    "notes": "오늘 다룰 내용 안내"
  },
  "content_slides": [
    {
      "title": "슬라이드 제목",
      "key_message": "핵심 메시지 한 문장",
      "layout": "text_and_visual",
      "bullets": ["포인트1", "포인트2", "포인트3"],
      "visual_description": "[막대 차트] 연도별 매출 추이 (2023~2025)",
      "notes": "발표자가 말할 내용"
    }
  ],
  "closing_slide": {
    "title": "감사합니다",
    "message": "Q&A",
    "contact": "이메일 등",
    "notes": "마무리 멘트"
  }
}
```

### Step 3: .pptx 파일 생성

JSON 파일을 저장한 뒤 스크립트를 실행한다:

```bash
python3 ~/.claude/skills/ppt-generator/scripts/generate_pptx.py <input.json> <output.pptx>
```

출력 파일은 사용자의 현재 작업 디렉토리 또는 데스크톱에 저장한다.

### Step 4: 자체 검증 체크리스트

파일 생성 후 아래 항목을 모두 확인하고 결과를 보고한다:

- [ ] **슬라이드 수 적정성**: 발표 시간 대비 슬라이드 수가 적절한가 (1장/1~1.5분)
- [ ] **필수 요소 완비**: 모든 슬라이드에 제목, 핵심메시지, 발표자노트가 있는가
- [ ] **시각자료 지시사항**: 시각자료가 필요한 슬라이드에 구체적 지시사항이 있는가
- [ ] **논리적 흐름**: 도입→전개→결론 구조가 자연스러운가
- [ ] **청중 적합성**: 용어 수준과 내용 깊이가 대상 청중에 맞는가
- [ ] **일관성**: 글꼴, 색상, 레이아웃이 통일되어 있는가
- [ ] **파일 생성 완료**: .pptx 파일이 정상적으로 생성되었는가

## 중요 원칙

- 시각자료 placeholder에는 구체적인 지시사항을 넣어서, 사용자가 나중에 실제 차트/이미지로 교체할 때 무엇을 넣어야 하는지 바로 알 수 있게 한다
- 발표자 노트는 실제로 말할 수 있는 자연스러운 문장으로 작성한다
- 핵심 메시지는 "이 슬라이드를 한 문장으로 요약하면?"에 대한 답이다
- 불릿 포인트는 3~5개로 제한한다. 너무 많으면 읽기 어렵다
