# Create Project (프로젝트 자동 생성기)

사용자가 "프로젝트 만들어줘"라고 하면, 폴더 생성부터 개발 준비 완료까지 전부 자동으로 처리하는 스킬이다.

## 입력 수집

사용자에게 **딱 2가지**만 물어본다 (이미 제공된 건 다시 묻지 않는다):

1. **프로젝트 이름** — 영문, 소문자, 하이픈(-) 사용. 예: `my-music-app`
   - 사용자가 한글로 말하면 적절한 영문 이름을 제안한다
2. **뭘 만들 건지** — 한 줄 설명. 예: "음악 메타데이터 생성 웹앱"

아래는 자동 판단한다 (물어보지 않는다):
- 기술 스택: 설명 내용에 맞게 자동 선택
- 폴더 위치: `~/Projects/` 아래에 생성 (없으면 만든다)

## 실행 순서

### Step 1: 프로젝트 폴더 생성

```bash
mkdir -p ~/Projects/{프로젝트이름}
cd ~/Projects/{프로젝트이름}
```

### Step 2: git 초기화

```bash
git init
```

### Step 3: .gitignore 생성

아래 내용으로 `.gitignore` 파일을 만든다:

```
node_modules/
.next/
dist/
build/
.env
.env.local
.DS_Store
*.log
```

### Step 4: 프로젝트 CLAUDE.md 생성

프로젝트 루트에 `CLAUDE.md` 파일을 만든다. 내용:

```markdown
# {프로젝트이름}

## 프로젝트 설명
{사용자가 말한 설명}

## 기술 스택
{자동 선택된 기술 스택}

## 폴더 구조
{생성된 폴더 구조}

## 빌드 & 실행
{기술 스택에 맞는 명령어}

## 규칙
- 코드 주석은 한국어로 작성
- 변수명은 영어, camelCase 사용
- 파일 수정 시 변경 내용 설명 포함
```

### Step 5: 기본 파일 생성

프로젝트 설명에 따라 적절한 기본 파일을 생성한다:

#### 웹페이지/웹사이트 → 단일 HTML
- `index.html` — 기본 구조 + 스타일 포함

#### 웹앱 (React/Next.js) → Next.js 프로젝트
```bash
npx create-next-app@latest . --typescript --tailwind --app --no-src-dir --no-eslint --import-alias "@/*"
```

#### Python 프로젝트 → 기본 구조
- `main.py` — 진입점
- `requirements.txt` — 의존성 목록

#### 판단 기준
| 사용자가 말한 내용 | 선택할 기술 |
|-------------------|-----------|
| "홈페이지", "랜딩페이지", "소개페이지" | 단일 HTML |
| "웹앱", "대시보드", "관리자페이지" | Next.js |
| "API", "서버", "봇", "자동화" | Python |
| "앱" (모호할 때) | Next.js |

### Step 6: 첫 커밋

```bash
git add -A
git commit -m "프로젝트 초기 설정

- 기본 파일 구조 생성
- CLAUDE.md 작성
- .gitignore 설정

Co-Authored-By: Claude <noreply@anthropic.com>"
```

### Step 7: 완료 보고

아래 형식으로 결과를 알려준다:

```
프로젝트 생성 완료!

📁 위치: ~/Projects/{프로젝트이름}
🛠 기술: {기술 스택}
📄 생성된 파일:
   - CLAUDE.md (프로젝트 설명서)
   - .gitignore (불필요 파일 무시 목록)
   - {기타 파일들}

💡 다음에 할 수 있는 것:
   - "메인 페이지 만들어줘"
   - "로그인 기능 추가해줘"
   - "디자인을 어둡게 바꿔줘"
```

## 중요 원칙

1. **질문 최소화**: 이름과 설명만 받고 나머지는 자동 판단
2. **바로 작업 가능한 상태**: 생성 직후 바로 "~해줘"라고 말하면 작업 가능
3. **세이브 포인트**: 첫 커밋을 자동으로 찍어서 언제든 초기 상태로 돌아갈 수 있게
4. **초보자 친화적**: 생성된 CLAUDE.md에 빌드/실행 명령어를 포함해서 나중에 헷갈리지 않게
