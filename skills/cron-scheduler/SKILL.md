## 의존성

| 의존성 | 종류 | 필수 여부 | 설명 |
|--------|------|----------|------|
| CronCreate 도구 | Claude Code 도구 | 필수 | 크론 작업 등록. Claude Code 내장 도구 (`CronCreate`, `CronList`) |
| 텔레그램 MCP | MCP | 선택 | 등록 완료 알림 전송용. 없으면 알림 기능 생략 |

## 역할

자연어로 말한 반복 스케줄을 크론 표현식으로 변환하고, CronCreate로 등록한 뒤 결과를 텔레그램으로 알린다.

## 트리거 조건

- "매주 ○요일에 ○○ 해줘"
- "매일 ○시에 ○○ 알려줘"
- "매월 ○일에 ○○ 해줘"
- "크론 등록해줘"
- "정기 작업 만들어줘"
- "크론 목록 보여줘"
- "등록된 스케줄 확인해줘"

## 자연어 → 크론 표현식 변환 규칙

크론 표현식 형식: `분 시 일 월 요일`

| 자연어 | 크론 표현식 |
|--------|------------|
| 매일 오전 9시 | `0 9 * * *` |
| 매주 월요일 오전 9시 | `0 9 * * 1` |
| 매주 금요일 오후 6시 | `0 18 * * 5` |
| 매월 1일 오전 10시 | `0 10 1 * *` |
| 매주 평일 오전 8시 | `0 8 * * 1-5` |
| 매시간 | `0 * * * *` |
| 매주 월수금 오전 9시 | `0 9 * * 1,3,5` |

요일 코드: 0=일, 1=월, 2=화, 3=수, 4=목, 5=금, 6=토

## 실행 절차

### 크론 등록 요청 시

1. 자연어에서 주기(분/시/일/주/월), 시각, 실행할 작업 추출
2. 크론 표현식 변환 및 사용자에게 확인 출력:
   ```
   등록할 내용:
   - 주기: {자연어 설명}
   - 크론: {표현식}
   - 작업: {실행할 내용}
   ```
3. CronCreate 도구로 등록
4. 등록 완료 후 텔레그램으로 알림 발송:
   ```
   크론 등록 완료
   - 이름: {작업명}
   - 주기: {자연어 설명}
   - 첫 실행: {다음 실행 시각}
   ```

### 크론 목록 확인 요청 시

1. CronList 도구 실행
2. 결과를 아래 형식으로 정리:
   ```
   등록된 크론 {N}개
   1. {작업명} — {주기} — {마지막 실행}
   2. ...
   ```

### 크론 삭제 요청 시

1. CronList로 목록 확인
2. 해당 크론 ID 특정
3. 삭제 실행 후 텔레그램 알림

## 사용자 자주 쓸 크론 패턴

등록 제안 가능한 정기 작업 예시:
- 매주 월요일 오전 9시: 주간 프로젝트 현황 브리핑 → 텔레그램 전송
- 매일 오전 8시 30분: Action Tracker 마감 임박 항목 확인 → 텔레그램 전송
- 매주 금요일 오후 5시: 주간 완료 항목 정리 → Obsidian 저장

## 텔레그램 알림 형식

크론 실행 결과 알림:
```
[크론 실행] {작업명}
시각: {실행 시각}
결과: {실행 결과 요약}
```

## 주의사항

- 크론 표현식 생성 후 반드시 자연어로 재확인 ("매주 월요일 오전 9시가 맞습니까?")
- 중복 크론 등록 방지: 등록 전 CronList로 유사 작업 확인
- 텔레그램 MCP가 연결된 상태에서만 알림 기능 작동
