# Remember Chat (대화 기억 시스템)

현재 대화의 핵심 내용을 요약해서 파일로 저장한다.
다음 대화에서 "지난번에 뭐 했지?" 라고 물으면 검색해서 찾아준다.

## 저장 위치

```
~/.claude/conversation-archive/
  index.md              # 전체 대화 목록 (최신순)
  2026/
    03/
      2026-03-22_주제명.md   # 개별 대화 요약
```

## 실행 모드

### 모드 1: 저장 (`/remember-chat` 또는 "대화 저장해")

현재 대화를 요약해서 저장한다.

#### Step 1: 대화 요약 생성

현재 대화에서 아래 정보를 추출한다:

```markdown
---
date: {YYYY-MM-DD}
topic: {대화 주제 한 줄}
project: {관련 프로젝트명, 없으면 "일반"}
tags: [{태그1}, {태그2}, {태그3}]
---

# {대화 주제}

## 뭘 했나
{핵심 작업 내용 3-5줄 요약}

## 결정사항
- {중요한 결정 1}
- {중요한 결정 2}

## 만든 것
- `{파일/기능1}` — {설명}
- `{파일/기능2}` — {설명}

## 미완료/다음에 할 것
- {아직 안 한 것}
- {다음에 이어서 할 것}

## 핵심 키워드
{나중에 검색할 때 쓸 키워드들: 기술명, 기능명, 문제 유형 등}
```

#### Step 2: 파일 저장

```bash
mkdir -p ~/.claude/conversation-archive/$(date +%Y/%m)
```

파일명: `{YYYY-MM-DD}_{주제를-영문-케밥케이스}.md`
예: `2026-03-22_superpowers-skill-creation.md`

#### Step 3: 인덱스 업데이트

`~/.claude/conversation-archive/index.md` 파일 맨 위에 추가:

```markdown
- [{날짜}] [{주제}]({상대경로}) — {한줄요약} `#{태그1}` `#{태그2}`
```

인덱스는 최신순으로 정렬. 200줄 초과 시 오래된 항목은 아래쪽에 유지하되 "더보기" 구분선 추가.

#### Step 4: 저장 완료 보고

```
대화 저장 완료!

- 파일: ~/.claude/conversation-archive/{경로}
- 주제: {주제}
- 태그: #{태그1} #{태그2}

다음 대화에서 "지난번에 {키워드} 어떻게 했지?" 라고 물어보면 찾아드립니다.
```

### 모드 2: 검색 ("지난번에 뭐 했지?", "이전에 ~한 적 있어?")

과거 대화를 검색해서 관련 내용을 찾는다.

#### Step 1: 아카이브 검색

```bash
# 키워드로 검색
grep -r "{키워드}" ~/.claude/conversation-archive/ --include="*.md" -l

# 태그로 검색
grep -r "#{태그}" ~/.claude/conversation-archive/ --include="*.md" -l

# 날짜로 검색
ls ~/.claude/conversation-archive/{YYYY}/{MM}/

# 프로젝트로 검색
grep -r "project: {프로젝트명}" ~/.claude/conversation-archive/ --include="*.md" -l
```

#### Step 2: 결과 요약

검색된 대화들을 요약해서 보여준다:

```
관련 대화 {N}개를 찾았습니다:

1. **[{날짜}] {주제}**
   {핵심 내용 1-2줄}
   → 자세히 보려면: "1번 자세히"

2. **[{날짜}] {주제}**
   {핵심 내용 1-2줄}
   → 자세히 보려면: "2번 자세히"
```

### 모드 3: 자동 제안

대화가 길어지고 의미 있는 작업이 진행됐으면, 대화 마무리 시점에 자동으로 제안:

```
이번 대화에서 꽤 많은 작업을 했네요. 저장해둘까요?
→ "ㅇㅇ" 또는 "/remember-chat"
```

## 중요 원칙

1. **프라이버시**: 개인정보, API 키, 비밀번호는 요약에 포함하지 않는다.
2. **핵심만**: 모든 대화를 기록하지 않는다. 결정사항과 결과물 중심으로.
3. **검색 가능성**: 태그와 키워드를 풍부하게 넣어서 나중에 찾기 쉽게.
4. **중복 방지**: 같은 날 같은 주제의 대화는 기존 파일에 추가(append).
5. **경량**: 각 요약은 30줄 이내로 유지. 코드는 포함하지 않는다.
