# Script Maker

## 핵심 규칙

사용자에게 터미널 명령을 직접 실행하라고 안내하는 대신, **항상 실행 가능한 .sh 스크립트 파일을 생성**한다.

## 언제 적용하는가

다음 상황에서 이 스킬이 작동한다:
- 권한 문제로 Bash 도구가 거부되었을 때
- 사용자가 직접 실행해야 하는 명령이 있을 때
- 여러 명령을 순서대로 실행해야 할 때
- `! command` 형태로 안내하려던 순간

## 작동 방식

1. `~/scripts/` 디렉토리에 스크립트 파일을 생성한다
2. 파일명은 작업 내용을 반영하는 짧은 영문 이름 (예: `fix-telegram.sh`, `setup-mcp.sh`)
3. 스크립트 상단에 한국어 주석으로 무엇을 하는 스크립트인지 설명한다
4. 사용자에게 실행 방법을 안내한다

## 스크립트 작성 규칙

```bash
#!/bin/bash
# ============================================
# [스크립트 설명 - 한국어]
# ============================================

set -e  # 에러 발생 시 중단

echo "=== 작업 시작 ==="

# 실제 명령들...

echo "=== 완료 ==="
```

## 안내 형식

스크립트 생성 후 사용자에게 이렇게 안내한다:

```
실행하세요:
! bash ~/scripts/파일명.sh
```

한 줄 이상 설명하지 않는다. 스크립트 안에 주석으로 이미 설명이 들어있다.

## 예외

다음은 스크립트로 만들지 않고 직접 안내해도 된다:
- `/reload-plugins` 같은 Claude Code 내부 명령
- `cd` 같은 단순 디렉토리 이동
- 대화형 입력이 필요한 명령 (예: `gcloud auth login`)
