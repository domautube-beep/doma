# session-journal 스킬

## 역할
매 세션 종료 시 자동으로 3곳에 기록하는 스킬.

## 트리거 조건
사용자가 다음 중 하나를 말할 때 즉시 실행한다:
- "끝"
- "저장"
- "마무리"
- "세션 끝"
- "/finish"
- CLAUDE.md 세션 프로토콜 종료 조건에 해당하는 발화

---

## 실행 순서

### Step 1: 세션 요약 생성

현재 대화 전체를 분석해서 아래 항목을 추출한다:

- **오늘 한 것** (3~5줄): 실제로 완료된 작업, 만든 것, 해결한 것
- **결정사항**: "~로 하기로 했다", "~는 보류", "~는 취소" 형태의 의사결정
- **미완료 / 다음에 할 것**: 언급됐지만 완료 안 된 것, 이어서 해야 할 것
- **변경된 파일 목록**: 읽거나 수정하거나 생성한 파일의 절대 경로

주제는 대화 내용을 가장 잘 요약하는 한국어 단어 1~3개로 정한다.
파일명용 kebab-case 영문 슬러그도 함께 만든다. (예: 주제 "세션 저널 스킬" → `session-journal-skill`)

---

### Step 2: 옵시디언 Sessions/ 폴더에 저장

**파일 경로:**
```
{OBSIDIAN_VAULT}/L4-Buffer/Sessions/YYYY-MM-DD_{슬러그}.md
```

날짜는 오늘 날짜(YYYY-MM-DD 형식)를 사용한다.

**파일 형식:**
```markdown
---
date: YYYY-MM-DD
tags: [세션, 주제태그1, 주제태그2]
---

# YYYY-MM-DD 세션: {주제}

## 한 것
- {항목1}
- {항목2}
- {항목3}

## 결정사항
- {결정1}
- {결정2}

## 미완료
- {항목1}
- {항목2}

## 변경된 파일
- `{파일경로1}`
- `{파일경로2}`
```

결정사항 또는 미완료 항목이 없으면 해당 섹션에 "없음"을 기재한다.

---

### Step 3: Action Tracker 업데이트

**파일 경로:**
```
{OBSIDIAN_VAULT}/Action Tracker.md
```

1. 파일을 Read 도구로 먼저 읽는다.
2. 세션에서 발생한 **새 액션 아이템**이 있으면 기존 내용 아래에 추가한다.
3. 세션에서 **완료된 항목**이 기존 목록에 있으면 `- [ ]`를 `- [x]`로 변경한다.
4. 기존 내용은 절대 삭제하지 않는다. 추가와 수정만 한다.

추가 형식:
```markdown
- [ ] {액션 내용} — {마감일 또는 "미정"} (출처: YYYY-MM-DD 세션)
```

---

### Step 4: Decision Log 업데이트

**파일 경로:**
```
{OBSIDIAN_VAULT}/Decision Log.md
```

1. 파일을 Read 도구로 먼저 읽는다.
2. 세션에서 내린 **의사결정이 있을 때만** 추가한다. 없으면 이 단계를 건너뛴다.
3. 기존 내용은 절대 삭제하지 않는다. 파일 맨 아래에 추가한다.

추가 형식:
```markdown
## YYYY-MM-DD — {결정 제목}
- **결정 내용**: {무엇을 어떻게 하기로 했는지}
- **이유**: {왜 이 결정을 내렸는지}
- **관련 세션**: [[L4-Buffer/Sessions/YYYY-MM-DD_{슬러그}]]
```

---

### Step 5: Pipe 교차 연결 체크

**Pipe 폴더 경로:**
```
{OBSIDIAN_VAULT}/01-Pipe/
```

1. `01-Pipe/` 폴더의 7개 Pipe 파일을 Read 도구로 읽는다.
2. 이번 세션에서 생성/수정된 파일, 결정사항, 액션 아이템을 기준으로 관련 Pipe를 찾는다.
3. 관련 Pipe가 있으면 해당 파일에 교차 연결(`[[Sessions/YYYY-MM-DD_{슬러그}]]` 또는 관련 노트 링크)을 1줄 이상 추가한다.
4. 관련 Pipe가 없으면 이 단계를 건너뛴다.

판단 기준:
- 세션에서 다룬 주제/프로젝트가 Pipe의 주제와 겹치는가
- 새로 만든 파일이 Pipe에서 다루는 영역에 해당하는가
- 결정사항이 Pipe의 방향성에 영향을 주는가

---

### Step 6: 인사이트 추출 (Curator)

이 세션에서 **새로 알게 된 것**을 추출한다:

1. **새 지식**: 이 세션에서 처음 등장한 개념, 도구, 사례, 숫자
   → 해당 Pipe의 "## 잠재 아이디어" 또는 L1-Knowledge/에 기록
2. **기존 지식 변경**: 이전에 알던 것이 바뀐 경우
   → 해당 Memory 파일 또는 Pipe 업데이트
3. **패턴 승격**: 반복되는 작업/판단이 프레임워크가 될 수 있는 경우
   → L2-Methods/에 신규 파일 또는 기존 파일에 추가
4. **Context Playbook 갱신**: SOUL.md/CLAUDE.md 규칙과 충돌하는 새 결정이 있으면
   → 해당 파일 업데이트 제안 (자율 범위 밖이면 사용자에게 물어봄)

없으면 "인사이트 없음"으로 건너뛴다.

---

### Step 7: Claude Memory 동기화

세션에서 발생한 내용 중 **미래 대화에서도 알아야 할 것**을 Claude memory에 반영한다.

**저장 경로:** `~/.claude/projects/-Users-{USER}/memory/` (심볼릭 링크 → {SYNC_DIR} → 집컴 자동 동기화)

**판별 기준:**

| 저장해야 할 것 | memory 타입 | 예시 |
|--------------|------------|------|
| 사용자가 "이렇게 해/하지 마" 피드백 | feedback | "에이전트 소환하지 마" |
| 프로젝트 핵심 결정/상태 변경 | project | "클레이 브랜딩 GO 판정" |
| 새로 알게 된 사용자 선호 | user | "PPT보다 HTML 선호" |
| 외부 시스템 위치/접근법 | reference | "사이니지 DB는 노션 X에" |

**저장하지 않을 것:** 일회성 태스크, 코드 변경 내용, Obsidian에만 있으면 충분한 상세 내용

**실행:**
1. MEMORY.md 읽기 → 기존 memory 중 업데이트할 것이 있는지 확인
2. 있으면 기존 파일 업데이트, 없으면 새 파일 생성
3. MEMORY.md 인덱스 업데이트
4. 해당 사항 없으면 이 단계 건너뛰기

---

## 완료 후 사용자에게 보고

6곳(Pipe + 인사이트 + Memory 포함) 기록이 끝나면 아래 형식으로 간단히 보고한다:

```
세션 저장 완료.

저장 위치:
- Sessions/YYYY-MM-DD_{슬러그}.md
- Action Tracker.md (신규 {N}개 / 완료 {M}개)
- Decision Log.md ({결정 N개 기록} 또는 "변경 없음")
- Pipe 교차 연결 ({업데이트한 Pipe 파일명} 또는 "해당 없음")
- 인사이트 ({추출 내용 한 줄} 또는 "없음")
- Claude Memory ({업데이트/생성한 파일} 또는 "해당 없음") → 집컴 자동 동기화
```

---

## 공통 규칙

- 모든 내용은 한국어로 작성한다.
- 파일을 수정하기 전에 반드시 Read 도구로 먼저 읽는다.
- 기존 파일 내용을 삭제하거나 덮어쓰지 않는다. 추가만 한다.
- {OBSIDIAN_VAULT} 루트 경로: `{OBSIDIAN_VAULT}/`
- 파일이 존재하지 않으면 새로 생성한다.
