# Knowledge Sync (Obsidian 지식 동기화)

## 의존성

| 의존성 | 종류 | 필수 여부 | 설명 |
|--------|------|----------|------|
| Obsidian Vault | 파일시스템 | 필수 | `{OBSIDIAN_VAULT}` 경로 — `L1-Knowledge/Domains/`, `01-Pipe/` 폴더 존재해야 함 |

대화 중 지식을 감지하거나 요청받으면 Obsidian vault에 자동 저장하고, 계층+자유 태그를 3개 이상 부여하여 연관 지식끼리 연결한다.

## 경로

```
VAULT = {OBSIDIAN_VAULT}
저장 폴더 = {VAULT}/L1-Knowledge/Domains/
Pipe 폴더 = {VAULT}/01-Pipe/
```

## 실행 모드

### 모드 1: 저장 (자동 감지 + 명시적 요청)

#### 트리거

**명시적**: "기억해", "저장해", "메모해", "이거 지식으로", "knowledge"
**자동 감지 기준** — 아래 중 하나 이상 해당하면 저장 제안:
- 새로운 개념/용어 정의가 등장
- 재사용 가능한 프레임워크/모델/원칙
- 외부 레퍼런스 (책, 영상, 도구, 사이트)
- 검증된 인사이트나 교훈
- 구체적 수치/데이터가 포함된 분석 결과

**자동 감지 시 행동**: 바로 저장하지 말고 먼저 제안한다.
```
이 내용 지식으로 저장할까요? → "{제목 초안}"
```
사용자가 승인하면 저장. "ㅇㅇ", "응", "저장해" 등 긍정이면 진행.

#### Step 1: 노트 생성

파일명: `{YYYY-MM-DD}-{제목-케밥케이스}.md`
위치: `{VAULT}/L1-Knowledge/Domains/`

```markdown
---
title: {제목}
created: {YYYY-MM-DD}
source: {출처 — 대화/책/영상/사이트/PDF 등}
tags: [{계층태그1}, {계층태그2}, {자유태그1}, ...]
related:
  - "[[관련 노트1]]"
  - "[[관련 노트2]]"
---

# {제목}

{본문 — 핵심 내용을 구조화하여 작성. 원문이 길면 요약.}

---
관련: [[관련 노트1]] | [[관련 노트2]]

#{태그1} #{태그2} #{태그3} ...
```

#### Step 2: 태그 부여 (최소 3개)

**계층 태그** — 기존 5대 MAP 기반, `대분류/세부` 형태:

| 대분류 | 세부 예시 |
|--------|----------|
| 디자인 | BX, UX, UI, 사이니지, 인쇄, 타이포, 컬러 |
| AI | Claude, Suno, MJ, ChatGPT, 프롬프트, 자동화, 워크플로우 |
| 음악 | 작곡이론, 작사, 프로덕션, 발매, 수익, 코드진행 |
| 사업 | SaaS, 마케팅, 전략, 수익모델, 브랜딩, 독립 |
| 인풋 | 책, 영상, 도구, 사이트, 아티클 |

**자유 태그** — 영역을 가로지르는 횡단 개념:
`수익화`, `전략`, `레퍼런스`, `프레임워크`, `인사이트`, `도구`, `벤치마킹`, `데이터`, `교훈`, `원칙`

**태그 부여 규칙**:
1. 계층 태그 최소 1개 + 자유 태그 최소 1개 = 합계 3개 이상
2. 내용의 핵심 주제와 활용 맥락을 모두 반영
3. 기존 태그 재사용 우선 — 새 태그는 기존에 없을 때만 생성
4. frontmatter `tags:` 배열과 본문 하단 `#태그` 인라인 양쪽 모두 기입

#### Step 3: MAP 연결

계층 태그의 대분류에 해당하는 MAP 파일에 링크 추가:

| 계층 대분류 | MAP 파일 |
|------------|---------|
| 디자인 | 01-Pipe/Pipe-디자인.md |
| AI | 01-Pipe/Pipe-AI무기.md |
| 음악 | 01-Pipe/Pipe-음악.md |
| 사업 | 01-Pipe/Pipe-사업.md |
| 인풋 | 01-Pipe/Pipe-인풋.md |

MAP 파일의 적절한 섹션 테이블에 행 추가:
```markdown
| {제목} | {핵심 인사이트 한 줄} | [[L1-Knowledge/Domains/{파일명}]] |
```

MAP에 맞는 섹션이 없으면 기존 섹션 중 가장 가까운 곳에 추가.

#### Step 4: 저장 완료 보고

```
지식 저장 완료!

- 파일: L1-Knowledge/Domains/{파일명}
- 태그: #태그1 #태그2 #태그3
- MAP 연결: {연결된 MAP명}
- 관련 노트: [[노트1]], [[노트2]]
```

### 모드 2: 소급 태깅 (`/knowledge-sync retag`)

기존 노트 전체를 순회하며 태그를 보강한다.

#### 대상 폴더 (순회 순서)

1. `L1-Knowledge/References/` — 레퍼런스, 분석 자료
2. `L2-Methods/` — 전략, 프로세스
3. `L3-Active/` — 프로젝트
4. `L4-Buffer/Sessions/` — 세션 기록
5. `01-Pipe/` — Pipe 파일 자체
6. 기타 (`L1-Knowledge/People/`, `L4-Buffer/Archive/Actions/`, `L4-Buffer/Archive/Decisions/` 등)

#### 처리 로직

각 노트에 대해:

1. **내용 읽기** — 본문 분석
2. **태그 생성** — 위 태그 체계에 따라 3개 이상
3. **frontmatter 처리**:
   - frontmatter 없음 → `---` 블록 새로 추가 (title, created, tags)
   - frontmatter 있고 tags 없음 → tags 필드 추가
   - frontmatter 있고 tags 있음 → 기존 태그 유지 + 새 태그 병합 (중복 제거)
4. **인라인 태그**: 본문 맨 하단에 `#태그` 추가 (이미 있으면 병합)
5. **related 필드**: 같은 태그를 공유하는 다른 노트가 있으면 연결

#### 진행 보고

5개 파일마다 중간 보고:
```
[12/34] 처리 중... 최근: {파일명} → #태그1 #태그2 #태그3
```

완료 시:
```
소급 태깅 완료!

- 총 처리: {N}개 노트
- 새 태그 추가: {N}개 노트
- 태그 병합: {N}개 노트
- 변경 없음: {N}개 노트

가장 많이 사용된 태그 TOP 5:
1. #태그 (N회)
2. ...
```

## 중요 원칙

1. **기존 내용 보존**: 노트 본문은 절대 수정하지 않는다. 태그/frontmatter만 추가/병합.
2. **태그 일관성**: 같은 개념에는 같은 태그. "AI음악"과 "AI/음악"이 혼재하면 안 됨.
3. **최소 3개**: 어떤 노트든 태그가 3개 미만이면 안 됨.
4. **연결 우선**: 태그의 목적은 분류가 아니라 연결. "이 지식이 어디에 쓰일 수 있는가"를 기준으로 태그.
5. **경량**: 지식 노트 본문은 50줄 이내 권장. 원본이 길면 핵심만 요약.
6. **프라이버시**: API 키, 비밀번호, 개인 금융정보는 저장하지 않는다.
