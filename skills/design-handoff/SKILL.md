# 디자인 핸드오프 문서 생성기

디자인 시안을 개발자/클라이언트에게 전달할 때 필요한 문서를 자동 생성한다.

## 문서 유형

### 1. 시안 설명서 (클라이언트 전달용)

```markdown
# [프로젝트명] 디자인 시안 설명서

**버전:** v1.0
**작성일:** [날짜]
**작성자:** 사용자 (하이컨시)

---

## 1. 디자인 콘셉트
[콘셉트 키워드 3-5개]
[콘셉트 설명 3-5줄]

## 2. 시안 구성
| No. | 페이지/화면 | 파일명 | 설명 |
|-----|-----------|--------|------|
| 1 | [페이지명] | [파일명] | [설명] |

## 3. 디자인 의도
### 컬러
- Primary: [컬러코드] — [사용 이유]
- Secondary: [컬러코드] — [사용 이유]

### 타이포그래피
- 제목: [폰트명] / [사이즈]
- 본문: [폰트명] / [사이즈]

### 레이아웃
[그리드/구조 설명]

## 4. 수정 요청 방법
- Figma 댓글로 수정 요청
- 수정 횟수: [N]회 포함
```

### 2. 개발 핸드오프 (개발자 전달용)

```markdown
# [프로젝트명] 개발 핸드오프

## 디자인 파일
- Figma: [링크]
- 에셋 폴더: [경로]

## 브레이크포인트
| 디바이스 | 너비 |
|---------|------|
| Mobile | 360px ~ 767px |
| Tablet | 768px ~ 1023px |
| Desktop | 1024px ~ 1440px |

## 컬러 토큰
| 토큰명 | 값 | 용도 |
|--------|-----|------|
| --color-primary | #XXXXXX | 주요 CTA |
| --color-text | #XXXXXX | 본문 텍스트 |

## 타이포 스케일
| 용도 | 폰트 | 사이즈 | 행간 | 자간 |
|------|------|--------|------|------|
| H1 | [폰트] | 32px | 140% | -0.02em |
| Body | [폰트] | 16px | 160% | 0 |

## 컴포넌트 목록
| 컴포넌트 | 상태 | 비고 |
|---------|------|------|
| [버튼] | Default/Hover/Disabled | [노트] |

## 인터랙션/애니메이션
| 요소 | 동작 | 이징 | 시간 |
|------|------|------|------|
| [요소] | [설명] | ease-out | 300ms |
```

### 3. 프로젝트 산출물 체크리스트

```markdown
# [프로젝트명] 산출물 체크리스트

## 디자인 파일
- [ ] Figma 파일 정리 (네이밍 통일)
- [ ] 컴포넌트 정리
- [ ] Auto Layout 적용
- [ ] 프로토타입 연결

## 에셋
- [ ] 아이콘 SVG 추출
- [ ] 이미지 최적화 (WebP)
- [ ] 로고 파일 (SVG/PNG/AI)
- [ ] 파비콘

## 문서
- [ ] 시안 설명서
- [ ] 스타일가이드
- [ ] 개발 핸드오프 문서
- [ ] 수정 이력
```

## 사용법

사용자가 Figma 링크를 주면 → Figma MCP로 디자인 정보 추출 → 문서 자동 생성
사용자가 구두로 설명하면 → 설명 기반으로 문서 틀 생성
